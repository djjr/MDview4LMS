# Project Specification: Context-Aware PDF & Question Viewer

## 1. Project Overview

Build a custom, continuous-scroll PDF viewer side-by-side with an iframe. The parent application must synchronize the iframe's URL source based on the currently viewed PDF page. Communication is strictly one-way: the parent app reads its own URL parameters to build a mapping, monitors the user's scroll position, and updates the iframe `src` accordingly.

## 2. Technical Stack

- **Frontend:** Vanilla HTML, CSS, and JavaScript. (Do not use heavy frameworks like React/Vue for this V1 prototype unless explicitly instructed).
    
- **PDF Engine:** `PDF.js` (Mozilla). Use CDN links for the library and the worker script.
    
- **Layout:** CSS Flexbox (2-column layout: Left = PDF, Right = Sidebar).
    

## 3. Core Architecture & Data Flow

### A. The URL Routing Schema

The application's state is driven entirely by the URL parameters of the parent window. The agent must parse standard URL queries using the `pageX=questionIds` format.

- **Example URL:** `https://parent-app.com/?page3=101,102&page5=104`
    
- **Logic:** When page 3 is active, the iframe loads `https://your-question-app.com/?q=101,102`.
    

### B. The Render Loop (`PDF.js`)

`PDF.js` must be used to render the document sequentially.

- The agent must fetch the document and loop through `pdf.numPages`.
    
- Crucially, `await` each page fetch to guarantee correct vertical DOM ordering.
    
- Each `<canvas>` generated by PDF.js must be wrapped in a `<div class="page-wrapper" data-page="X">` element. This wrapper is the target for scroll tracking.
    
- _Note:_ Text-selection layers are out of scope for V1. Render as a purely visual canvas.
    

### C. Scroll Tracking (Intersection Observer)

Do not use `window.onscroll` event listeners. Use the `IntersectionObserver` API for performance.

- **Target:** Watch the `.page-wrapper` elements.
    
- **Threshold:** Trigger when a page wrapper is `0.5` (50%) visible in the viewport.
    
- **Debouncing (Critical):** Implement a `400ms - 500ms` debounce timer inside the observer callback. If the user scrolls rapidly past multiple pages, the iframe must _not_ update until scrolling settles, preventing flickering and excessive network requests.
    

## 4. Stepwise Implementation Plan

Please implement this project following a stepwise refinement approach. Do not attempt to build the entire script in a single pass.

**Phase 1: Scaffolding & Layout**

- Create `index.html`, `style.css`, and `app.js`.
    
- Set up a full-height Flexbox layout: `#pdf-viewer` (scrollable, flex: 1) and `#sidebar` (fixed width, holding `#question-iframe`).
    
- Mock the iframe to point to `about:blank`.
    

**Phase 2: URL Parsing & State Setup**

- Write a function to instantiate `URLSearchParams` and parse the parent window's query string.
    
- Create a helper function `updateSidebar(pageNumber)` that constructs the target iframe URL based on the extracted parameters and handles the fallback state (when a page has no assigned questions).
    

**Phase 3: The PDF.js Engine**

- Import `PDF.js` via CDN and configure the `GlobalWorkerOptions`.
    
- Write the `async` render loop to fetch a sample PDF, create the wrapper divs with `data-page` attributes, generate the canvases, and append them to the DOM in strict sequential order.
    

**Phase 4: The Intersection Observer**

- Initialize the `IntersectionObserver` with the correct root (the scrollable container) and threshold.
    
- Attach the observer to the wrapper divs during the Phase 3 render loop.
    
- Implement the intersection logic with the `setTimeout` debounce mechanism.
    
- Connect the observer output to the `updateSidebar(pageNumber)` function from Phase 2.
    

## 5. Agent Constraints & Guardrails

- **Keep it decoupled:** The parent app must not attempt to send `postMessage` events into the iframe or read data out of it. It strictly acts as a router updating the `src` attribute.
    
- **Handle Async carefully:** Ensure the DOM wrappers are created synchronously in the `for` loop, even if the canvas drawing takes a few milliseconds longer, to prevent pages rendering out of order.
    
- **Placeholder Variables:** Use `const PDF_URL = 'sample.pdf'` and `const IFRAME_BASE_URL = 'https://example-questions.com/'` at the top of your script so they are easily swappable.
